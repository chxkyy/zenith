# 合作机构管理中心 - 产品规格说明（第一批）

## Summary

构建合作机构管理中心，首批实现**合作机构管理池**与**管理人准入**两大核心模块。支持按需创设内部合作机构池、查看机构企业信息与合作产品清单；同时提供完整的管理人在线准入流程，涵盖评分评估、准入材料上传、内部 OA 流程对接及移动端审批能力。

## Problem

当前缺乏统一的合作机构管理平台，机构信息分散、管理人准入流程线下化效率低、评分标准不统一，导致：
- 合作机构信息无法集中维护和快速检索
- 管理人准入材料散落在各处，审批链路不透明
- 尽调评分依赖人工计算，易出错且难以追溯
- 无法有效跟踪机构的生命周期状态（在池 / 调出 / 过期）

## Goals

- 建立合作机构统一档案：企业基本信息、合作产品清单、扩展属性一站式管理
- 实现管理机构池的灵活创设与维护：支持多维度分类、状态追踪
- 提供管理人全流程在线准入：从申请 → 材料提交 → 评分 → 审批 → 入池
- 支持评分体系可配置化，为后续尽调模板差异化打下基础
- 对接内部 OA 审批流，支持移动端审批体验

## Non-goals

- **本批次不包含**：1.1.3 管理人协议管理、1.1.4 尽职调查与质量分析评分的完整多模板体系、1.1.5 扩展信息管理、1.1.6 过期与调出池管理（后续批次覆盖）
- 不涉及对外部机构的数据对接或 API 集成
- 不涉及 AI 自动评分或智能推荐
- OA 对接以接口预留为主，具体 OA 系统集成由后续批次完成

## Behavior

### 一、合作机构管理池（1.1.1）

#### 1.1 机构池列表与检索

1. 系统提供「合作机构管理池」主页面，默认展示所有已创建的机构池卡片/列表。
2. 每个机构池展示以下摘要信息：
   - 池名称（必填）
   - 池类型/分类标签（如「公募池」「私募池」「专户池」等）
   - 池内机构数量
   - 最近更新时间
   - 当前状态（启用 / 停用）
3. 支持以下检索条件：
   - 按池名称模糊搜索
   - 按池类型筛选
   - 按状态筛选（全部 / 启用 / 停用）
4. 检索结果实时响应，分页展示（遵循项目分页规范）。
5. 无匹配结果时显示空状态提示：「未找到符合条件的机构池」。

#### 1.2 机构池的增删改查

6. **创建机构池**：
   - 用户点击「新建机构池」按钮，弹出创建表单
   - 必填项：池名称（唯一性校验，不可与已有池重复）、池类型
   - 可选项：描述说明、关联负责人
   - 创建成功后跳转至该机构池详情页
   - 创建者自动拥有该池的管理权限（受数据权限策略控制）

7. **编辑机构池**：
   - 池名称修改后需做唯一性校验
   - 池类型变更需记录变更日志
   - 已停用的机构池不允许修改池内机构数据（仅允许重新启用或编辑基础信息）

8. **停用/启用机构池**：
   - 启用状态的池可被停用；停用后池内机构数据保留但不可新增/修改机构
   - 停用状态的池可被重新启用
   - 停用操作需二次确认：「停用后该池内的机构将暂停使用，确认停用？」

9. **删除机构池**：
   - 仅空池（池内机构数为 0）且处于停用状态时可删除
   - 非空池删除时提示：「该池内仍有 N 家机构，请先移除所有机构后再删除」

#### 1.3 机构池详情 — 企业信息

10. 进入机构池后展示「池内机构列表」，列出该池下所有合作机构。
11. 每家机构展示企业名片信息：
    - 机构全称（必填）
    - 机构简称
    - 统一社会信用代码（必填，格式校验：18 位）
    - 机构类型（如基金公司 / 资管公司 / 证券公司 / 其他）
    - 成立时间
    - 注册资本
    - 法定代表人
    - 注册地址
    - 联系电话 / 联系邮箱
    - 机构Logo（图片上传）
12. 支持在企业列表页直接搜索：按机构名称、统一社会信用代码模糊检索。
13. 点击某机构进入**机构详情页**，展示完整的「企业信息」Tab 内容。

#### 1.4 机构池详情 — 合作产品清单

14. 机构详情页提供「合作产品清单」Tab，展示该机构下的所有合作产品。
15. 每条产品记录包含：
    - 产品名称（必填）
    - 产品代码
    - 产品类型（如公募基金 / 专户产品 / 私募产品 / 其他）
    - 合作状态（合作中 / 已终止 / 暂停）
    - 合作起始日期
    - 产品经理/联系人
16. 支持添加、编辑、删除合作产品记录。
17. 删除已关联的产品时需确认：「确认从该机构下移除此产品记录？此操作不可撤销。」
18. 产品合作状态变更为「已终止」时，系统记录终止日期和操作人。

#### 1.5 机构入池与移出

19. **入池操作**：
    - 在机构池详情页点击「添加机构」，弹出选择器
    - 可从「全部机构库」（系统内已有机构记录）中选择机构入池
    - 若机构不存在于系统内，可选择「新建机构并入池」同步创建
    - 同一机构可同时存在于多个机构池中（多对多关系）
    - 入池时可选填入池备注、入池日期（默认当天）

20. **移出池操作**：
    - 在池内机构列表中点击「移出」，弹出确认框
    - 确认后机构从当前池中移除（不影响该机构在其他池中的归属，也不删除机构本身）
    - 记录移出操作日志：操作人、时间、目标机构、原所属池

### 二、管理人准入（1.1.2）

#### 2.1 准入流程总览

21. 管理人准入采用**标准化工作流**模式，典型流程为：
    ```
    发起准入申请 → 填写基本信息 → 上传准入材料 → 内部评分 → 提交审批 → OA/移动端审批 → 准入通过 → 自动入指定机构池
    ```
22. 所有参与准入流程的管理人记录统称为**准入申请单**，每个申请单有唯一编号（格式：`ADM-YYYYMMDD-NNNN`）。
23. 申请单在整个生命周期中的状态流转如下：

```
草稿(DRAFT) → 待评分(PENDING_REVIEW) → 待审批(PENDING_APPROVAL)
    → 已通过(APPROVED) → 已拒绝(REJECTED) → 已撤销(WITHDRAWN)
```

24. 状态流转规则：
    - `DRAFT` → `PENDING_REVIEW`：申请人提交材料后自动流转
    - `PENDING_REVIEW` → `PENDING_APPROVAL`：评分完成后自动流转
    - `PENDING_APPROVAL` → `APPROVED`：审批人通过后流转
    - `PENDING_APPROVAL` → `REJECTED`：审批人驳回后流转
    - 除 `DRAFT` 和 `REJECTED` 外，申请人可主动撤销（→ `WITHDRAWN`）
    - `APPROVED` 和 `REJECTED` 为终态，不可再变更

#### 2.2 准入申请 — 基本信息

25. 发起准入入口：
    - 从「合作机构管理池」页面点击「发起准入」
    - 或从机构详情页点击「申请准入」
26. 申请表单的基本信息部分包含：
    - **管理人名称**（必填）：机构或个人名称
    - **管理人类型**（必填）：公募 / 私募 / 保险 / 银行理财子 / 其他
    - **统一社会信用代码**（必填，机构类必填）
    - **注册资本**
    - **成立日期**
    - **法定代表人/负责人**
    - **注册地址**
    - **联系人**（必填）
    - **联系电话**（必填）
    - **联系邮箱**
    - **目标入池**（必填）：选择要进入的目标机构池（单选或多选，取决于池配置）
27. 表单支持**暂存草稿**功能：填写过程中可随时保存，保存后可在「我的申请」列表中继续编辑。
28. 必填项未完成时，提交按钮置灰并提示「请完善必填信息后再提交」。

#### 2.3 准入申请 — 准入材料上传

29. 准入材料区分为**标准材料清单**（根据管理人类型动态展示）和**自定义补充材料**两部分。所有材料项均为**选填**，不强制要求上传即可提交评审。
30. 标准材料清单包含以下类别（根据管理人类型有差异）：
    - **营业执照/登记证书**（图片/PDF）
    - **金融许可证/备案证明**（PDF）
    - **最近一期财务报表**（PDF）
    - **公司章程**（PDF）
    - **主要人员资质证明**（PDF/图片包）
    - **内部控制制度文件**（PDF）
    - **其他监管要求的文件**（PDF）
31. 材料上传要求：
    - 单个文件大小限制：不超过 50MB
    - 支持格式：PDF、JPG、PNG、Word（.doc/.docx）、Excel（.xls/.xlsx）
    - 每个材料项可上传多个文件（如分多页扫描的执照）
    - 上传后可预览、替换、删除单个文件
32. 自定义补充材料：申请人可额外上传不在标准清单中的材料，需填写材料名称和说明。
33. 所有材料均为选填，用户可在零材料情况下直接点击「提交评审」（系统给出提示「您尚未上传任何材料，确认提交？」以二次确认）。

#### 2.4 准入评分

34. 申请提交后进入**待评分**状态，由指定的内部评分人员进行评分操作。
35. 评分界面展示：
    - 申请人填写的全部基本信息（只读）
    - 全部已上传的材料（可在线预览）
    - **评分表单**（根据配置的评分模板展示）
36. 本批次的评分采用**简化版通用评分模板**（完整的多模板差异化为 1.1.4 批次实现），包含以下评分维度：
    - **公司综合实力**（权重 20%）：规模、行业地位、股东背景
    - **合规与风控**（权重 25%）：监管记录、内控体系、历史违规情况
    - **投研能力**（权重 25%）：团队履历、投研流程、历史业绩
    - **服务能力**（权重 15%）：响应速度、客户服务质量
    - **扣分项**（权重 15%）：负面舆情、重大风险事件
37. 每个评分维度采用 **100 分制**，评分人给出 0-100 的整数分数。
38. 系统根据各维度权重自动计算**加权总分**（保留 1 位小数）。
39. 评分人可填写**评语/备注**（选填，不超过 500 字）。
40. 评分完成后点击「提交审批」，申请单流转至待审批状态。若评分低于预设的**准入分数线**（默认 60 分，可配置），系统给出警告：「当前评分 XX 分低于准入线 YY 分，是否仍提交审批？」，评分人确认后方可提交。
41. 评分人在提交前可**退回补正**：将申请退回给申请人，附上退回原因（必填）。申请回到 `DRAFT` 状态，申请人收到通知后可补充材料重新提交。

#### 2.5 审批流程与 OA 对接

42. 评分通过后的申请进入**审批队列**，按优先级和时间排序展示。
43. 审批人（可配置单人审批或多级会签）在审批列表中查看待审批事项。
44. 审批详情页展示：
    - 全部基本信息
    - 全部材料和评分结果（含各维度得分、加权总分、评语）
    - 审批操作区域
45. 审批操作选项：
    - **通过**：填写审批意见（选填）→ 申请变为 `APPROVED`
    - **驳回**：必须填写驳回原因（必填，不少于 10 字）→ 申请变为 `REJECTED`
    - **退回补正**：填写退回原因 → 申请回到 `DRAFT` 状态
46. **OA 对接接口**：
    - 系统预留 `ApprovalService` 接口，定义标准的审批回调方法
    - 支持 OA 系统通过 HTTP 回调推送审批结果
    - 本批次以**模拟审批模式**运行（内置审批流），OA 实际对接在后续批次完成
47. **移动端适配**：
    - 准入申请详情页、评分页、审批页均需适配移动端浏览
    - 材料预览支持移动端常用格式（PDF 在线预览、图片缩放）
    - 审批操作按钮在移动端尺寸下保持足够触摸区域（最小 44px）

#### 2.6 准入通过后的自动处理

48. 申请审批通过（`APPROVED`）后，系统执行以下自动处理：
    - 将该管理人作为新机构记录写入系统（若尚不存在）
    - 根据「目标入池」字段的选择，将该机构加入对应的机构池
    - 设置机构的初始合作状态为「合作中 - 准入通过」
    - 向申请人发送准入通过通知（站内消息 / 邮件，本批次先实现站内消息）
49. 若目标机构池设置了**自动分配负责人**规则，则按规则分配该机构的负责人员。
50. 审批拒绝（`REJECTED`）后：
    - 向申请人发送拒绝通知，包含驳回原因
    - 申请单归档至「已拒绝」列表，保留 90 天后可手动清理

#### 2.7 我的申请与查询

51. 每个用户可在「我的准入申请」页面查看自己发起的所有申请单。
52. 列表展示：申请单号、管理人名称、目标池、当前状态、提交时间、最近操作时间。
53. 支持按状态筛选（全部 / 草稿 / 评审中 / 审批中 / 通过 / 驳回 / 已撤销）。
54. 对于 `DRAFT` 状态的申请单，用户可继续编辑或删除。
55. 对于非终态的申请单，用户可查看详情但不能编辑（只读模式）。
